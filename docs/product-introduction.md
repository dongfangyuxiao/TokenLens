# TokenLens 产品介绍文档

## 产品概述

**TokenLens（探云令）** 是一款面向企业研发团队的自动化代码安全审计平台。通过深度集成大语言模型（LLM）、Semgrep 静态分析与 OpenSCA 软件成分分析，对代码仓库的每一次提交进行智能安全审计，实时发现注入漏洞、认证绕过、供应链投毒等高危风险，并通过多种渠道即时告警。

### 核心价值

- **自动化审计**：无需人工介入，自动监控所有代码提交
- **智能分析**：结合 AI 大模型与静态分析工具，准确识别安全风险
- **实时告警**：通过钉钉、飞书、企业微信等渠道即时通知
- **全生命周期管理**：从漏洞发现到修复跟踪的完整闭环

## 核心功能

### 1. 多模式智能审计

#### 增量扫描（Poison Scan）
- **目标**：实时监控供应链投毒和恶意代码注入
- **范围**：所有分支的新增与修改文件
- **特点**：专注检测后门植入、typosquatting、异常外联等供应链攻击
- **推荐频率**：每小时

#### 增量审计（Incremental Audit）
- **目标**：对最近提交进行深度安全审计
- **范围**：变更文件的 LLM 深度分析
- **特点**：只上报严重与高危风险，降低噪音
- **推荐频率**：每天（工作日下班后）

#### 全量审计（Full Audit）
- **目标**：建立完整安全基线
- **范围**：main/master 主分支完整历史
- **特点**：覆盖所有漏洞级别，定期全面检查
- **推荐频率**：每周

### 2. 多模型交叉审计

支持配置多个 AI 模型协同工作，分阶段去噪：

- **审计模型**：初步发现潜在风险
- **检查模型**：过滤误报和低风险项
- **验证模型**：最终确认高危漏洞

典型组合：DeepSeek 审计 + GPT 检查 + Claude 验证

### 3. 多平台代码源接入

| 平台 | 支持方式 |

| GitHub | Token + Organization |
| GitLab / 腾讯工蜂 / 阿里云效 | Token + 自定义域名 |
| Bitbucket | App Password |

**自定义**
- 任意 OpenAI 兼容接口

### 5. 即时告警与通知

支持多种通知渠道：
- 钉钉（支持签名验证）
- 飞书 / 企业微信
- Slack
- 邮件（SMTP，支持 SSL）

### 6. 应急排查能力

- **SCA 组件清单**：软件成分分析，检测恶意依赖
- **全代码库检索**：支持关键词模糊搜索（如 `swagger.html`）
- **漏洞统计分析**：按天/周/月查看趋势

### 7. 白名单与误报管理

- 对已确认误报的漏洞加入白名单
- 后续扫描自动过滤
- 支持按仓库 + 文件 + 漏洞标题精确匹配

### 8. 漏洞生命周期管理

每条漏洞支持状态流转：
- 新增 → 修复中 → 已修复 / 暂不处理
- 审计历史全程可查

### 9. Syslog 实时同步

登录、扫描、漏洞发现等关键事件实时转发至外部 Syslog 服务器（支持 UDP / TCP），便于接入 SIEM。

### 10. 自定义 Prompt

内置多语言专项提示词：
- Frontend、Backend、Smart Contract
- Java、PHP、Python、Go、Node.js、C#、Ruby、Rust、C/C++

支持按文件扩展名匹配，可在 Web 界面自由编辑或新增。

## 漏洞检测范围

### 前端安全
- XSS（存储型、反射型、DOM 型）
- 原型链污染
- 敏感数据泄露
- 不安全存储
- CSP 配置缺失
- CORS 错误配置
- postMessage 未校验来源
- 开放重定向
- 点击劫持
- 供应链投毒

### 后端安全
- SQL / NoSQL / 命令注入
- XXE（XML 外部实体注入）
- 认证与授权缺陷（JWT、IDOR、OAuth2）
- 批量赋值
- 敏感信息泄露
- 弱加密
- SSRF（服务端请求伪造）
- 文件路径穿越
- 不安全反序列化
- CSRF
- 业务逻辑漏洞
- 速率限制缺失

### 依赖与供应链
- 恶意代码注入
- 后门植入
- typosquatting（域名抢注）
- 混淆代码
- 异常外联请求
- 数据外泄逻辑
- 已知 CVE 漏洞（通过 OpenSCA 检测）

## 产品授权

TokenLens 采用**机器码 + 授权文件**的离线授权机制：

### 授权特点
- **离线验证**：无需联网，授权文件本地校验
- **数字签名**：基于 Ed25519 算法，防篡改
- **功能控制**：可按功能点精确授权
- **有效期管理**：支持设置授权到期时间

### 授权流程
1. 客户在系统设置中查看并导出机器码
2. 供应商根据机器码签发授权文件（`license.json`）
3. 客户上传授权文件即可激活
4. 系统自动校验签名和有效期

### 功能点控制
- `poison_scan`：投毒检测
- `incremental_audit`：增量审计
- `full_audit`：全量审计
- `instant_analysis`：即时分析

## 部署方式

### 快速部署（一键脚本）

```bash
# 解压部署包
tar -xzf tokenlens-YYYYMMDD.tar.gz -C /tmp/

# 进入解压目录
cd /tmp/tokenlens

# 运行一键部署脚本
sudo bash deploy/deploy.sh
```

### Docker 部署

```bash
docker compose up -d --build
```

### 系统要求

**操作系统**
- Ubuntu 20.04 / 22.04 / 24.04 LTS（推荐）
- Debian 11 / 12

## 使用流程

### 1. 配置代码平台账户
进入**代码平台**页面，添加 GitHub / GitLab 等账户的 Token，填写 Owner（组织或用户名）。

### 2. 配置 LLM
进入 **AI 配置**页面，选择 LLM 提供商并填入 API Key 和 Model。

### 3. 配置审计计划
进入**系统设置**页面，为三类计划分别添加执行时间点。

### 4. 配置告警通知
进入**通知渠道**页面，添加钉钉、飞书、企业微信或邮件配置。

### 5. 查看审计结果
- **扫描历史**：查看每次扫描的漏洞汇总
- **漏洞详情**：按严重程度、状态筛选
- **漏洞统计**：按天/周/月查看趋势图

### 6. 即时审计
在**即时检测**页面粘贴代码片段或上传文件（支持 ZIP），立即触发审计。
