# TokenLens 探云令
## 企业级代码安全审计平台

**让代码安全审计更智能、更高效**

---

## 产品定位

TokenLens 是一款面向企业研发团队的**自动化代码安全审计平台**

### 核心价值
- ✓ **自动化**：无需人工介入，自动监控所有代码提交
- ✓ **智能化**：AI 大模型 + 静态分析，准确识别安全风险
- ✓ **实时化**：即时发现漏洞，多渠道告警推送
- ✓ **全流程**：从漏洞发现到修复跟踪的完整闭环

---

## 核心功能

### 三种审计模式

| 模式 | 目标 | 频率 |
|------|------|------|
| **增量扫描** | 供应链投毒检测 | 每小时 |
| **增量审计** | 高危漏洞深度分析 | 每天 |
| **全量审计** | 完整安全基线 | 每周 |

**任务针对性扫描**：每类任务在 Prompt 层面即明确扫描目标，LLM 从分析阶段就针对性工作

---

## 多模型交叉审计

### 三段式协同验证

```
审计模型 → 检查模型 → 验证模型
(发现)     (过滤)     (确认)
```

### 典型组合
- DeepSeek 审计 + GPT 检查 + Claude 验证
- 分阶段去噪，降低误报率
- 自动沉淀为 adaptive skills

## 漏洞检测范围

### 前端安全
XSS、原型链污染、敏感数据泄露、CSP 配置缺失、CORS 错误配置、开放重定向、点击劫持、供应链投毒

### 后端安全
SQL/NoSQL/命令注入、XXE、认证授权缺陷、SSRF、文件路径穿越、不安全反序列化、CSRF、业务逻辑漏洞

### 供应链安全
恶意代码注入、后门植入、typosquatting、混淆代码、异常外联、已知 CVE

### 静态扫描

## 产品授权

### 离线授权机制
- **机器码 + 授权文件**
- 无需联网验证
- 基于 Ed25519 数字签名，防篡改

### 功能点控制
- `poison_scan`：投毒检测
- `incremental_audit`：增量审计
- `full_audit`：全量审计
- `instant_analysis`：即时分析

### 授权流程
客户导出机器码 → 供应商签发授权文件 → 客户上传激活

## 部署方式

### 一键部署
```bash
tar -xzf tokenlens-YYYYMMDD.tar.gz -C /tmp/
cd /tmp/tokenlens
sudo bash deploy/deploy.sh
```

### Docker 部署
```bash
docker compose up -d --build
```

### 系统要求
- **操作系统**：Ubuntu 20.04+ / Debian 11+ / CentOS 7+
- **硬件**：2 核 / 2 GB（最低）、4 核 / 8 GB（推荐）
- **Python**：3.10+（推荐 3.11/3.12）

### 5. 查看审计结果
扫描历史、漏洞详情、漏洞统计
