# 交付与防抄袭建议

## 目标

降低“源码被直接复制后复用”的风险，优先交付可运行产物，不直接交付完整源码。

## 推荐交付方式（优先级从高到低）

1. 私有镜像仓库交付（推荐）
- 只提供 Docker 镜像拉取权限，不提供源码仓库权限。
- 镜像仓库启用账号审计、IP 白名单、短期访问令牌。

2. 私有部署代运维
- 由你方统一部署，客户仅使用服务，不接触源码与镜像构建细节。

3. 源码交付（仅在必须时）
- 必须签署 NDA + 许可协议（明确禁止再分发、禁止反编译、禁止二开转售）。
- 仓库采用最小权限、按时回收账号、开启审计日志。

## 强制控制清单

- 使用私有 Git 仓库，关闭匿名访问。
- 启用分支保护、强制评审、提交签名（可选）。
- 提交前运行 secret guard，避免令牌泄漏。
- 生产环境令牌全部使用最小权限且定期轮换。
- 报告链接要求登录鉴权（已在系统内支持）。

## 交付前检查

```bash
pre-commit run --all-files
git status
```

确认无 `.env`、数据库、报告、同步快照等敏感文件进入提交。
