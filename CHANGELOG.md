# 更新日志

## 2026-03-26

### ✨ 新功能

#### 跨文件联合安全分析（二次分析）

在原有逐文件并发分析的基础上，新增跨文件第二轮联合分析，能够发现单独看任意一个文件都察觉不到的漏洞。

**工作机制**

```
第一轮（并发）             第二轮（串行）
──────────────             ──────────────
file_a.py → LLM ──┐
file_b.py → LLM ──┼──→ 汇总摘要 + 代码片段 + 已发现漏洞 ──→ LLM ──→ 跨文件漏洞
file_c.py → LLM ──┘
```

**触发条件**
- 同一仓库/提交包含 2 个或以上文件
- 已配置 LLM（无 LLM 时跳过）

**专门识别的跨文件风险类型**

| 类型 | 示例 |
|---|---|
| 数据流跨文件 | 用户输入在 `routes.py` 进入，未经过滤直接到达 `db.py` 中的 SQL 执行点 |
| 鉴权缺口 | 路由定义在文件 A，鉴权中间件在文件 B，二者之间存在可绕过的路径 |
| 配置联动 | `config.py` 关闭了 SSL 验证，`client.py` 中的调用因此可被中间人攻击 |
| 协同修改 | 攻击者同时修改校验函数和调用处，绕过安全检查 |
| 跨文件提权链 | 低权限操作经由多个文件传递，最终触达高权限函数 |

**Token 用量控制**
- 每个文件的代码片段截取前 1600 字符
- 超过 15 个文件时，多余文件仅附文件摘要、不附代码片段
- 要求 LLM 不重复报告第一轮已发现的问题

**涉及文件**
- `analyzer.py`：新增 `_analyze_cross_file()` 函数；`_analyze_single_file()` 额外返回文件摘要；`analyze_commit()` 在第一轮结束后调用第二轮
- `database.py`：`findings` 表新增 `is_cross_file`（0/1）和 `cross_files`（JSON 数组）两列，含自动迁移

**UI 展示**

跨文件漏洞在漏洞详情弹窗中显示紫色 **🔗 跨文件** 标签，并以标签形式列出所有涉及的文件，而非单一文件路径。

#### 扫描触发弹窗（选 AI 模型）

点击顶栏"投毒检测 / 增量审计 / 全量审计"按钮后，弹出模态窗口，而非直接触发扫描。

弹窗内容：
- 扫描类型的图标、名称、功能描述（支持中英双语）
- AI 模型选择器（可选"全局 AI 配置"或任意已配置的 AI 配置列表）
- 取消 / 开始扫描按钮

移除了之前顶栏的 LLM 下拉框，交互更清晰。

---

#### 报告导出：DOCX 和 PDF 格式

扫描历史表格新增两个导出按钮：

| 按钮 | 实现方式 |
|---|---|
| **PDF** | 在新标签页打开 HTML 报告，自动触发浏览器打印对话框，另存为 PDF，完整支持中文，无需额外字体库 |
| **DOCX** | 后端用 `python-docx` 生成结构化 Word 文档，含封面信息、按仓库分组的漏洞列表、严重程度颜色标注，原生支持中文 |

DOCX 导出接口：`GET /api/scans/{id}/export?format=docx`

### 🐛 修复 / 优化

- `_analyze_single_file()` 返回值从 `list` 改为 `(findings, summary)` 元组，为跨文件分析收集文件摘要提供基础
- `save_findings()` 更新 INSERT 语句，保存 `is_cross_file` 和 `cross_files` 字段
- 数据库迁移：已有数据库启动时自动追加 `is_cross_file` 和 `cross_files` 列，无需手动变更
